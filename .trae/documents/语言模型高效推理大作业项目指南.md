## 项目目标与成果物
- 目标：在不改参数、不训练的前提下，对 Pythia-70M 在长上下文场景的推理进行加速与显存节省，完成可复现实验与报告。
- 数据与评测：在 `wikitext` 与 `pg-19` 上进行 perplexity（PPL）与加速（tokens/s、显存）测试；`pg-19` 选取单一超长样本即可。
- 算法范围：基线 Dense（不压缩）与若干 KV 压缩方法（建议 StreamingLLM、SnapKV、PyramidKV，若时间允许可增加 ExpectedAttention/KNorm 作为对比）。
- 交付物：独立公开 GitHub 仓库（代码+README），包含复现所需的脚本、配置与结果。

## 硬件与软件要求
- 最低硬件：NVIDIA CUDA GPU（显存≥6GB）；CPU 亦可跑但速度较慢，且长上下文易 OOM。
- 推荐硬件：NVIDIA 10GB–24GB 显存（如 3080/3090/4090/A6000），便于 `pg-19` 长样本与多算法对比。
- 注意事项：KV Cache 占用与序列长度线性相关；长序列（>16k tokens）在 Dense 基线下可能逼近 6–8GB 显存，压缩后显存占用显著下降。
- 软件环境：Python 3.10+，PyTorch(CUDA)、Transformers、Datasets、Accelerate、`kvpress`。可选安装 FlashAttention 与评估扩展包以获得更好的速度与现成评估工具。

## 集成 KVPress 与算法实现
- 使用方式：导入后自动注册 `kv-press-text-generation` 管线；可直接传入不同 `press` 实例与压缩参数。
- 建议实现：
  - `StreamingLLMPress`：保留序列头+尾的窗口，适合极长上下文；参数为窗口大小与保留策略。
  - `SnapKVPress`：依据最近查询的平均注意力进行打分裁剪；对通用场景有效。
  - `PyramidKVPress`：分层金字塔式保留不同密度的 KV，兼顾远近信息。
  - 可选：`ExpectedAttentionPress` 或 `KnormPress` 作为对照。
- 参数维度：
  - 预填充压缩：`compression_ratio`（例如 0.3–0.9）
  - 解码期压缩（实验性）：周期压缩与目标缓存大小（若使用，需选择 ScorerPress 作为 `base_press`）

## 评测方案与指标
- 指标：
  - 精度：PPL（`wikitext` 主评），`pg-19` 可报告生成一致性或简单困惑度近似
  - 性能：吞吐（tokens/s）、端到端延迟、显存峰值（MB/GB）
  - 压缩：`compression_ratio` 或目标缓存大小，记录被保留/裁剪的比例
- 设计：
  - 对每种 `press` 与多组参数做网格搜索（如 0.3/0.5/0.7/0.9）
  - 在相同上下文与生成长度下比较 Dense 与压缩的差异
- 输出：统一写入 CSV/JSON，并保存关键日志与可视化（曲线：PPL vs 压缩率、显存 vs 序列长度、吞吐 vs 压缩率）。

## PPL 评测参考实现
- 思路：滑窗累积负对数似然，计算跨文档平均困惑度；长文本采用分片避免 OOM。
- 伪代码：
```
import torch, math
from datasets import load_dataset
from transformers import AutoModelForCausalLM, AutoTokenizer

def compute_ppl(model_name, dataset_name, split, stride, max_length, device):
    tok = AutoTokenizer.from_pretrained(model_name)
    model = AutoModelForCausalLM.from_pretrained(model_name).to(device)
    data = load_dataset(dataset_name, split=split)
    nll_sum, tok_count = 0.0, 0
    for ex in data:
        text = ex["text"]
        ids = tok(text, return_tensors="pt").input_ids[0]
        for i in range(0, len(ids), stride):
            end = min(i + max_length, len(ids))
            input_ids = ids[i:end].unsqueeze(0).to(device)
            labels = input_ids.clone()
            labels[:, :-1] = -100
            with torch.no_grad():
                out = model(input_ids, labels=labels)
            nll_sum += out.loss.item() * (end - i - 1)
            tok_count += (end - i - 1)
    ppl = math.exp(nll_sum / max(tok_count, 1))
    return ppl
```
